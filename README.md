# Slack Digest

A personal daily digest widget — surfacing unanswered DMs, saved messages, and team news from Slack. Runs as a local HTML file; optionally auto-refreshes via a background server powered by Claude Code.

## What it does

- **Today tab** — action items: unanswered DMs, saved messages, Jira tickets waiting on you. Cards expand to show context and a link back to Slack. Mark items Done to dismiss them (persists across reloads).
- **Community tab** — team news: announcements, social moments, open questions in your channels.
- **Growth tab** — static tips for working with AI tools.
- **Refresh button** — if the server is running, triggers a live Claude Code session that checks Slack, Gmail, and Google Calendar, then rewrites `data.js`.

---

## Quick start (no server, static data)

1. Clone or download this folder.
2. Copy `data.example.js` → `data.js` and edit it with your own cards.
3. Open `index.html` in a browser. Done.

The Refresh button will show "Server offline" until you set up the server — that's fine for static use.

## Full setup (live auto-refresh)

### Prerequisites

- **Node.js 18+**
- **Claude Code CLI** — install at [claude.ai/code](https://claude.ai/code) or `npm install -g @anthropic-ai/claude-code`
- **Claude Code MCPs authenticated** — the regeneration script uses:
  - `mcp__slack` — Slack search and thread reading
  - `mcp__google-calendar` — calendar events
  - `mcp__google-mail` — Gmail sent mail
  - `mcp__atlassian` — Jira ticket status
  
  Run `claude /mcp` in your terminal and authenticate each one before first use.

### Step 1 — Create your regeneration prompt

```bash
cp regenerate-prompt.template.md regenerate-prompt.md
```

Edit `regenerate-prompt.md` and replace every `YOUR_NAME` with your full name (e.g. `Alex Kim`) and `YOUR_EMAIL` with your work email. This file is gitignored — it stays local to you.

### Step 2 — Seed initial data

```bash
cp data.example.js data.js
```

Edit `data.js` with a few real cards, or run the regeneration script manually to let Claude populate it from scratch:

```bash
bash regenerate.sh
```

This runs a Claude Code session (takes 1–3 minutes on first run). Check `regenerate.log` if anything goes wrong.

### Step 3 — Start the server

```bash
node server.mjs
```

The server runs at `http://127.0.0.1:3141`. Keep it running in a terminal tab. The Refresh button in the widget will now trigger live data pulls.

### Step 4 (optional) — Run automatically on login

Install the two LaunchAgents so the server starts at login and data refreshes hourly on weekdays.

**Prepare the plist files:**

```bash
# Copy templates
cp com.YOURNAME.slack-digest.plist.template        com.YOURUSERNAME.slack-digest.plist
cp com.YOURNAME.slack-digest-server.plist.template com.YOURUSERNAME.slack-digest-server.plist

# In both files, replace:
#   YOUR_HOME      →  /Users/yourusername  (your home directory)
#   YOUR_REPO_PATH →  /path/to/this/repo  (absolute path to this repo)
#   com.YOURNAME   →  com.yourusername    (a unique reverse-DNS label)
```

**Install:**

```bash
LAUNCHAGENTS=~/Library/LaunchAgents

cp com.YOURUSERNAME.slack-digest.plist        "$LAUNCHAGENTS/"
cp com.YOURUSERNAME.slack-digest-server.plist "$LAUNCHAGENTS/"

launchctl load "$LAUNCHAGENTS/com.YOURUSERNAME.slack-digest.plist"
launchctl load "$LAUNCHAGENTS/com.YOURUSERNAME.slack-digest-server.plist"
```

To uninstall:

```bash
launchctl unload "$LAUNCHAGENTS/com.YOURUSERNAME.slack-digest.plist"
launchctl unload "$LAUNCHAGENTS/com.YOURUSERNAME.slack-digest-server.plist"
rm "$LAUNCHAGENTS/com.YOURUSERNAME.slack-digest.plist"
rm "$LAUNCHAGENTS/com.YOURUSERNAME.slack-digest-server.plist"
```

## Files

| File | Description | Committed? |
|---|---|---|
| `index.html` | The widget UI — all CSS and JS inline | Yes |
| `server.mjs` | Local refresh server on port 3141 | Yes |
| `regenerate.sh` | Shell script that calls `claude -p` to regenerate data.js | Yes |
| `regenerate-prompt.template.md` | Template for the Claude prompt — copy to `regenerate-prompt.md` and fill in your name/email | Yes |
| `regenerate-prompt.md` | Your personalized prompt (gitignored) | No |
| `data.example.js` | Example data showing the card schema | Yes |
| `data.js` | Your live digest data — generated by regenerate.sh (gitignored) | No |
| `dismissed.json` | IDs of cards you've dismissed (gitignored) | No |
| `*.plist.template` | LaunchAgent templates — fill in paths and install to ~/Library/LaunchAgents | Yes |
| `*.plist` | Your personalized LaunchAgents (gitignored) | No |

## Troubleshooting

**Refresh button says "Server offline"** — run `node server.mjs` first.

**Regeneration produces no output / wrong output** — check `regenerate.log`. Most common cause: Claude Code MCPs not authenticated. Run `claude /mcp` and auth Slack + Google Calendar + Gmail + Atlassian.

**Cards not disappearing after Done** — dismissals persist in `localStorage` and `dismissed.json`. If you move to a new machine, start fresh.

**LaunchAgent not running** — check `~/Library/Logs/com.YOURUSERNAME.slack-digest.log` or run `launchctl list | grep digest`.
